# 파파존스 판교2테크노밸리점 모바일 키오스크

현재 배포 버전은 `kiosk V8`입니다. 프로젝트 운영·배포·보안·변경 내역은 앞으로 이 파일 하나에서 관리합니다.

## 주요 화면

- `index.html`: 고객용 모바일 키오스크
- `admin.html`: 주문 관리자 화면
- `seats.html`: 좌석 현황 화면
- `admin-login.html`: 관리자 로그인
- `assets/images/`: 피자·사이드·음료·초기화면 이미지
- `i18n/`: 한국어·영어·일본어·중국어·베트남어 문구

## 실행

정적 웹 서버의 루트를 이 폴더로 지정한 뒤 `index.html`을 엽니다. 모바일 화면은 아이폰 세로모드 기준으로 확인합니다.

## Firebase

1. `firebase-config.js`에서 대상 Firebase 프로젝트 설정을 확인합니다.
2. Firebase Console의 Firestore Rules에 `firestore.rules` 내용을 적용합니다.
3. 관리자 화면에서 알림음을 활성화한 뒤 테스트 주문을 진행합니다.
4. 주문 접수·조리 시작·조리 완료와 좌석 상태 변경을 확인합니다.

실제 운영에서는 Firebase Authentication 관리자 권한, App Check, 예산 알림과 사용량 알림을 함께 적용하는 것을 권장합니다. 웹 API Key 자체가 보안 경계는 아니며 실제 접근 제어는 Firestore Rules와 Authentication이 담당합니다.

## 보안 주의사항

- 운영 환경에서 공개 읽기·쓰기 테스트 규칙을 사용하지 않습니다.
- 주문·좌석·줄서기·고객·통계 접근 권한은 `firestore.rules`를 기준으로 관리합니다.
- 전화번호는 필요한 최소 정보만 저장하며 관리자 화면의 외부 데이터는 HTML 이스케이프를 유지합니다.
- 가격 위변조 방지를 강화하려면 Cloud Functions에서 공식 메뉴 가격으로 주문 금액을 재계산하는 방식을 권장합니다.

## 버전 변경 내역

### V8

- 언어 선택 카드 영역을 화면 중앙에 배치하고 좌우 여백을 동일하게 조정

- 장바구니 상품 정보를 `사이즈 - 크러스트 - 도우 - 구성 - 피자` 순으로 정리
- 한판 주문의 `한판` 문구를 숨기고 반반 주문에만 `반반` 표시
- 기본 도우의 고객 표시 명칭을 `클래식`으로 변경
- 모든 한국어 `반반` 문구를 `반반`으로 통일
- 초기화면 로고 20% 축소, 투명 배경과 좌측 상단 배치 적용
- 로고 아래 매장명을 `판교2테크노밸리점`으로 축약
- 우측 상단 언어 버튼을 `LANGUAGE`로 변경
- 주문 접수 완료 아이콘을 파파존스 빨간색으로 변경
- 기존 크러스트 선택 화면과 주문 진행 로직은 유지
